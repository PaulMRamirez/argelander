# PHASE-5: State authorities

Subject: retire the idea that Argelander is an SGP4 tool with extras. TLE and SGP4 are one state authority among many, the only one the engine computes in process from a compact string, which is why they dominate the current prose rather than the design; the seam (SPEC-PROVIDER) is source-agnostic and already accepts states from any authority that can express position and velocity in a body-fixed frame at a series of epochs. This phase broadens the inbound-state authorities and rebalances the documentation to match, and it is governed throughout by ADR-0014, the frame boundary: the core stays body-fixed at the seam, any inertial-to-body-fixed rotation lives only in a provider and never above the rendering-grade level, interchange parsers refuse inertial input by default, and the budget for an allowed rotation is one arcminute of arc. The four input paths the survey names in priority order, SPICE first, then elements, then interchange, then live telemetry (section 9.2, AGE-04), become the spine of both the code and the guide.

Deliver: an OMM parser (`parseOmm`) that decodes CCSDS Orbit Mean-elements Messages in KVN and XML and the flat Celestrak and Space-Track JSON into the mean elements the existing SGP4 path consumes, landing beside `parseTle` in `argelander-providers`, so a Celestrak or Space-Track element export feeds `Sgp4Provider` unchanged, with a thin, optional, credential-free retrieval adapter or documented recipe for the two catalogs where Celestrak is a public GET and Space-Track authentication is the host's, the library accepting at most a session token the host already obtained and never a username or password, all of it outside the core (ADR-0014 rule 7); an OEM reader that decodes CCSDS Orbit Ephemeris Messages, the tabulated state vectors with their interpolation metadata and their time system converted to Et, into the pre-sampled path as a sibling of the CSV, CZML, and GeoJSON providers, body-fixed only and refusing an inertial reference frame with a named error that points the caller at the SPK sampling service (ADR-0014 rules 3 and 5), so precision low-Earth-orbit, deep-space, suborbital, and Space-Track special-perturbations ephemerides all render through the existing `PresampledProvider` interpolation; the SPICE-first path made concrete as a documented recipe plus a thin reference sampler over the existing `httpStateProvider` wire that emits body-fixed states from a SPICE backend (spiceypy or WebGeocalc), with no embedded CSPICE and no new server package (ADR-0002), so the charter's primary state authority is exercised end to end rather than only named; the live-telemetry provider shape written down as a contract but not built here, the seam a Yamcs bridge will implement in PHASE-3, so that phase is a binding rather than a fresh design; and the documentation reframed around the seam, the guide's provider coverage and the README leading with the StateProvider taxonomy and the four-path priority rather than with SGP4, SGP4 presented as the zero-backend near-Earth demonstration it is, and a suborbital sounding-rocket example, a stratospheric-balloon example, and a planetary-orbiter example added so a reader sees the breadth of platforms the seam already serves.

Exit: a Celestrak element export, both the GP JSON and the OMM form, parses through `parseOmm` and renders the same ground track as the object's TLE through `Sgp4Provider`; an OEM document renders a strip through the pre-sampled path, and an OEM in an inertial frame is refused with the named error that names the SPK sampling service rather than drawn as a confidently wrong footprint; the reference SPK sampler serves body-fixed states over the HTTP wire and a demo or test renders a planetary or Earth track from it; the guide and the README lead with the seam and the SPICE-first priority and carry the suborbital, balloon, and planetary examples, with TLE and SGP4 no longer predominant; the live-telemetry provider contract is recorded for PHASE-3 to implement; and the charter holds throughout, `argelander-core` still ships zero runtime dependencies, the frozen strip schema and SPEC-PROVIDER do not move, no path introduces propagation beyond the existing SGP4, orbit determination, frames math above the rendering-grade provider level of ADR-0014, or an embedded CSPICE, and any addition that fixes a new wire shape, the live-telemetry push contract in particular, lands with its own ADR before implementation.
